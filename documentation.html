File Structure

Everything is plain HTML, CSS, and JS — no build tools, no Node, no bundler. Upload files directly to GitHub and they're live.

💡 How it works

GitHub Pages serves any index.html inside a folder at that folder's URL. No configuration — just create the folder, put an index.html in it, push to main.

GitHub Pages Setup

1
Create the repository
Go to github.com/new. Repo name must be exactly ekans-0.github.io. Set to Public.
2
Enable Pages
Settings → Pages → Source: Deploy from branch → Branch: main → Folder: / (root) → Save.
3
Upload files
Drag all HTML files into the repo via github.com, or use git push. Live within 60 seconds.

Git workflow

bash
# one-time setup
git clone https://github.com/ekans-0/ekans-0.github.io
cd ekans-0.github.io

# after any change
git add .
git commit -m "add experience: JHU research"
git push origin main
# → live in ~60 seconds

  <iframe

⚠ Size limit

GitHub has a 100MB per-file limit. Keep videos under 20MB. Use HandBrake (free) to compress, or use YouTube embedding for longer content.

Add a New Page

1
Create the folder
Create blog/ (or whatever the URL should be).
2
Copy a page as template
Copy projects/index.html — it already has nav, loader, cursor, bg-grid, footer, and all the CSS vars. Change the breadcrumb and class="active" nav link.
3
Add nav link to every page
Find the .n-links nav in index.html, projects/index.html, debate/index.html, experience/index.html and add <a href="/blog/">blog</a> to all of them.
4
Push and done
No config needed. The new path is live within 60 seconds of pushing.

Design System

All colors are CSS variables at the top of every page's <style> block:

css variables
--gold:  #e4cf94;  /* primary accent — warm gold */
--teal:  #4dd9c0;  /* secondary — running/live indicators */
--red:   #e8677a;  /* error / loss / negative */
--ind:   #7c8cf8;  /* ML / indigo */
--grn:   #5cdb8e;  /* success / positive */
--t1:    #eae8e3;  /* primary text */
--t2:    #6e7a94;  /* body text */
--t3:    #323849;  /* muted */
--bdr:   rgba(228,207,148,.07);  /* default border */
--bdr2:  rgba(228,207,148,.20);  /* emphasized border */

Change these in any page's :root to retheme that page. Keep them consistent across all pages for a unified look.

Canvas Widgets

The hero widget on the homepage has three tabs: Double Pendulum, Brownian Motion, and Fourier Series. All are live canvas animations.

To add a new tab: add a .cw-tab button and a corresponding .cw-panel with a canvas, then write the animation in JS following the same pattern. The switchWidget(id, btn) function handles tab switching automatically.

To replace the featured project canvas (neural net), just replace the <canvas id="nn-c"> with an <img> or <video> — the container styling handles the rest.

// Edit on GitHub directly

Navigate to any file in your repo → click the pencil icon → edit in browser → scroll down → "Commit changes". No local setup required. This is the fastest way to make small edits.

// Custom domain

Want ekanshdave.com? Buy a domain (~$10/yr), create a CNAME file in your repo root with just the domain name, then set up DNS: Settings → Pages → Custom domain.
